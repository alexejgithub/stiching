# Touch/stylus input parity

Type: grilling
Status: resolved

## Question

Does editing need full touch parity (finger, phone/tablet), desktop-only with responsive viewing elsewhere, or something in between?

## Answer

Stylus input must have full parity with mouse for every tool (draw, select, rotate, move, mirror) — precision matters and a stylus is precise enough to match mouse interactions. Bare-finger touch is not required to be fully tuned, only to not break (no horizontal-scroll disasters, app remains usable/viewable). Full finger-first touch optimization is a candidate for a later pass, not a v1 requirement.
